# OctoAcme Project Management Docs

Welcome to the OctoAcme project management process documentation. This README serves as your central entry point to discover, understand, and apply OctoAcme's project management practices.

## Overview of OctoAcme Project Management

OctoAcme operates on a structured lifecycle approach to project delivery, emphasizing customer value, iterative delivery, clear ownership, and data-informed decisions. The organization follows five distinct phases: **Initiation**, **Planning**, **Execution**, **Release**, and **Close & Retrospective**. Each phase builds on validated outcomes and maintains a clear audit trail through documentation.

### Key Principles
- **Customer-first**: Prioritize customer value and usability
- **Iterative delivery**: Deliver small, testable increments
- **Clear ownership**: Each project has a named Project Manager (PM) and Product Lead
- **Data-informed decisions**: Measure impact and iterate based on evidence
- **Psychological safety**: Encourage feedback and learning

### Core Roles
The organization maintains clear role definitions to prevent bottlenecks and ensure accountability:
- **Project Manager (PM)**: Coordinates delivery, schedules, risk, and communications
- **Product Manager (PdM)**: Defines outcomes, prioritizes backlog, and measures success
- **Developers**: Implement features, collaborate on design and testability
- **QA/Testing**: Validate quality and acceptance criteria
- **Stakeholders**: Provide inputs and approvals

### Key Artifacts
- Project Charter / One-pager
- Roadmap and Release Plan
- Sprint/Iteration Backlog
- Acceptance Criteria & Definition of Done
- Risk Register
- Retrospective notes and action items

### Communication Cadence
- Daily standups (15 minutes) — focus on progress, blockers, dependencies
- Weekly sync between PM + PdM
- Twice-weekly standups for delivery team (or as agreed)
- Monthly stakeholder updates
- Ad-hoc escalations as needed

## OctoAcme Process Documents

### 1. [Project Management Overview](octoacme-project-management-overview.md)
**Purpose**: Concise introduction to OctoAcme's approach, roles, and key artifacts

Provides a shareable guide for new teammates to understand how OctoAcme runs projects. Covers core roles, key artifacts, the high-level lifecycle, and communication cadence.

### 2. [Project Initiation Guide](octoacme-project-initiation.md)
**Purpose**: Define initial steps to validate and authorize work, align stakeholders, and create a lightweight plan

Use this when a new project idea or feature proposal is ready to be explored. Includes the Project One-pager template and an initiation checklist that covers problem statement, goals, success metrics, stakeholder alignment, and decision gates.

### 3. [Project Planning](octoacme-project-planning.md)
**Purpose**: Turn an approved initiative into an actionable plan and backlog for delivery

Guides the breakdown of work into shippable increments, identification of dependencies and risks, and alignment of timelines and responsibilities. Includes backlog item templates, sprint planning guidance, and risk & dependency management workflows.

### 4. [Execution & Tracking](octoacme-execution-and-tracking.md)
**Purpose**: Guidance for managing day-to-day execution and tracking progress toward project milestones

Defines team rhythm (standups, weekly delivery syncs, demos), Pull Request workflow conventions, quality & testing standards, and blocker escalation paths. Includes an execution checklist covering branching, CI, demos, and risk tracking.

### 5. [Release & Deployment Guide](octoacme-release-and-deployment.md)
**Purpose**: Standardize how OctoAcme releases features to production to reduce risk and improve observability

Covers release types (patch, minor, major), pre-release requirements, deployment checklist, and rollback & incident playbook. Includes a release notes template for communicating changes to stakeholders.

### 6. [Risk Management & Communication](octoacme-risks-and-communication.md)
**Purpose**: Explain how to identify, manage, and communicate risks and dependencies

Provides the Risk Register template and lifecycle, stakeholder communication strategies with templates, and a three-level escalation path (team → PM → Product Lead → Sponsor). Includes incident communication best practices.

### 7. [Retrospective & Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md)
**Purpose**: Capture learnings and convert them into actionable improvements

Guides running effective retrospectives, tracking action items, and building a continuous improvement culture. Includes structure for retrospectives, an action item template, and guidance on measuring impact of improvements.

### 8. [Roles & Personas](octoacme-roles-and-personas.md)
**Purpose**: Define typical roles and responsibilities used in OctoAcme project docs

Provides detailed persona definitions for Developers, Product Managers, and Project Managers, including their responsibilities, goals, and typical communication patterns. These personas frame scenarios and sample interactions throughout the OctoAcme system.

## How to Use These Docs

1. **New to OctoAcme?** Start with [Project Management Overview](octoacme-project-management-overview.md) for a high-level introduction.

2. **Starting a new project?** Follow the sequence: [Project Initiation Guide](octoacme-project-initiation.md) → [Project Planning](octoacme-project-planning.md) → [Execution & Tracking](octoacme-execution-and-tracking.md).

3. **Need guidance on a specific phase?** Use the table of contents above to jump to the relevant document.

4. **Managing risks or communicating updates?** Refer to [Risk Management & Communication](octoacme-risks-and-communication.md).

5. **Preparing for release?** Check [Release & Deployment Guide](octoacme-release-and-deployment.md).

6. **End of sprint or project?** Run a retrospective using [Retrospective & Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md).

7. **Clarifying roles?** See [Roles & Personas](octoacme-roles-and-personas.md) for detailed responsibilities.
